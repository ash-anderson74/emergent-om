# Changelog

All notable changes to EmergentOS are documented in this file.

EmergentOS is a living specification. Changes reflect learning, clarification, and evolution rather than “feature delivery”.

---

## [v0.1] — Initial Public Specification

### Added
- Core EmergentOS intent, scope, and positioning
- Explicit rejection of deterministic planning as a default
- Outcome-driven planning principles (Lean Value Trees, OKRs)
- Modular operating system framing
- Core domains:
  - Direction & Purpose
  - Leadership & Culture
  - Product & Value Creation
  - Flow & Enablement
  - Learning & Sensing
- Quarterly cadence with embedded slowification
- Coaching-led governance model
- Adaptive funding and portfolio flow concepts
- Alignment with DevOps (Three Ways) and Lean principles
- Role patterns:
  - Value Outcome Lead (VOL)
  - Team Outcome Lead (TOL)
  - Architecture Outcome Lead (AOL)
- Explicit non-goals and anti-patterns

### Notes
This version represents an initial synthesis intended to invite critique, learning, and refinement. Language, structure, and emphasis are expected to change.

---

## Versioning Guidelines

- **Major versions (v1.0, v2.0):**  
  Structural or philosophical shifts that materially change the operating model

- **Minor versions (v0.x):**  
  Clarifications, refinements, modular additions, or re-framing without breaking core principles
